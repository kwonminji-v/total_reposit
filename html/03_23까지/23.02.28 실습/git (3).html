1. 작업 폴더 추적되는 않는 Untracked files (작업폴더에 있기 때문에 아직은 추적되지 않음) --> 한번도 버전관리를 하지 않은 파일이라고 부른다. --> 나의 폴더에만 있기때문에 어딘가의 공간으로 이동이 되어야 추적되는 파일이 됨 2. 작업폴더를 무데에 올리는 <스테이지>가 필요 staging area는 commit을 하기 전에 commit할 파일들을 골라놓는 곳입니다. 그리고 staging area에 파일을 넣는 행위를 staging이라고 한다. git add 명령어로 staging 할 수 있다. 양식 : **git add . 을 명령어로 입력하게 되면 모든 unstage된 파일들을 전부 스테이징 할 수 있다. 아래 명령어는 add와 함께 사용이 가능한 명령어이다. -f (force) : 기존 중복 디렉터리를 무시하고 추가한다. -p (partial) : 수정한 부분에 대해서만 추가한다. -v (verbose) : 자세한 실행 과정과 결과를 출력한다. 3. repository는 commit된 파일의 버전들을 모아놓는 곳입니다. repsitory의 실체를 구경하고 싶다면 작업 폴더안에 숨겨져 있는 .git 폴더를 열어보면 된다. git하고 commit (덩어리를 만들었다는 의미) 가장 많이 사용되는 명령어 중 하나로, 수정한 파일을 추가한 후, 수정 사항에 대해 기록을 남긴다. commit을 진행하면 꼭 아래와 같이 간단한 메세지를 넣어줘야 한다 -m 옵션으로 명령과 동시에 커밋 메시지를 추가한다. git commit -m '첫파일을 만들었음' 4.git status 상태창 양식 5.git log --all --online commit된 기록을 한눈에 파악하고 싶으면 해당 명령어를 사용한다. 양식 아래 명령어는 log와 함께 사용이 가능한 명령어이다. --graph : 브랜치나 태그된 소스 트리를 그림으로 표현하여 출력한다. --grep＝키워드 : 키워드가 포함된 로그만 출력한다. git push 이전에 $ git remote add origin https://github.com/minji6313.git 6.git push -u 원격저장소주소 main (-u 옵션은 방금 입력한 주소를 기억해 두라는 뜻) -로컬저장소의 main 브랜치를 원격 저장소에 올리라는 뜻입니다. 다른 브랜치도 올릴 수 있음 / 다음부터는 주소를 길게 입력 안하고 git push만 입력해도 잘 됨 원격 repository 주소는 이렇게 https:// 부터 시작해서 .git으로 끝난다 이후에는 git push만 해도 기억해서 들어갈 수 있게 된다.
